Bring the Universe to You

The Basics

We assembled an embeddable widget for any website in which with some simple configuration, you can view PO.DAAC data in a 2d lineplot, or a 3d map plot.

To embed this service in your page, it is simple. First thing is first, you include one JavaScript file in a webpage. 

<script data-main="//	.jpl.nasa.gov/widgets/plot/0.9/js/main.js" src="//cds.jpl.nasa.gov/widgets/plot/0.9/components/requirejs/require.js"></script>

From there, I can add an interactive widget like this to my webpage in one more line of code :)

Option 1: Embedding a Plot in HTML

We have two options to embed a widget in a site. Either programatically through JavaScript, or through a standard HTML tag.

The HTML tag approach (Recommended if you're not a JavaScript person) 

<div 
	data-viz-type="plot.xyz" 
	xdata="http://podaac-w10n.jpl.nasa.gov/w10n/allData/amsre/L3/sst_1deg_1mo/tos_AMSRE_L3_v7_200206-201012.nc/lon[]" 
	xmask="[100000002004087730000]" 
	xlabel="lon" 
	ydata="http://podaac-w10n.jpl.nasa.gov/w10n/allData/amsre/L3/sst_1deg_1mo/tos_AMSRE_L3_v7_200206-201012.nc/lat[]" 
	ylabel="lat" 
	zdata="http://podaac-w10n.jpl.nasa.gov/w10n/allData/amsre/L3/sst_1deg_1mo/tos_AMSRE_L3_v7_200206-201012.nc/tos[2:3]"
	zmask="[100000002004087730000]" 
	zlabel="sea surface temperature" 
	format="spherical"
	colorbar="#ffffe5,#fff7bc,#fee391,#fec44f,#fe9929,#ec7014,#cc4c02,#993404,#662506"></div>
Wait, wait, I'm Definitely a JavaScript person, show me the API...

HTML Embed Attributes

All XYZ HTML Embed Parameters:

xdata= webified URL string for x data
ydata = webified URL string for y data
zdata = webified URL string for z data
xlabel|ylabel|zlabel = label for x|y|z axis label
xmask|ymask|zmask = string (as a js array) i.e. xmask="[14,25,35]". Values not to plot

format = string, format of the object. Possible values: spherical|sphericalterrain|flat|flatterrain. Default is Flat.

Easter Eggs:
rotating = boolean (string i.e. rotating="true"), default is false (whether or not to slowly auto rotate your model);
riftview = boolean (string, i.e. riftview="true"), default is false (style your widget fullscreen with CSS, and view your model an Oculus Rift!! Requires a chrome plugin: https://github.com/benvanik/vr.js)

Option 2: Embedding a Plot Widget with the JavaScript API

First we take care of making some containers for these widgets. You can customize them however you'd prefer. In this case we just append two widgets with jQuery. 

var xyz = $('<div id="xyzexample" />');

$('body').append(xyz);

Next, we add metadata

Now we need to give the widget some metadata 

//xyz widget meta
var xyz_params = {
    type:'xyz',
    selector:'#xyzexample',
    xdata:'http://podaac-w10n.jpl.nasa.gov/w10n/allData/amsre/L3/sst_1deg_1mo/tos_AMSRE_L3_v7_200206-201012.nc/lon[]',
    ydata:'http://podaac-w10n.jpl.nasa.gov/w10n/allData/amsre/L3/sst_1deg_1mo/tos_AMSRE_L3_v7_200206-201012.nc/lat[]',
    zdata:'http://podaac-w10n.jpl.nasa.gov/w10n/allData/amsre/L3/sst_1deg_1mo/tos_AMSRE_L3_v7_200206-201012.nc/tos[0:1]',
    zmask:[100000002004087730000],
    zlabel:'zlabel',
    ymask:[100000002004087730000],
    xlabel:'xlabel',
    ylabel:'ylabel',
    format:'spherical',
    width:300,
    height:200
}

Now, call the widget

And finally, trigger the widget programatically when we need to. 

$(document).trigger('callPlotAPI',xyz_params);

All XYZ Plot Parameters

var xyz_params = {
	type:'xyz', //STRING, REQUIRED. THIS SHOULD ALWAYS BE 'xyz'
	selector:'#xyzexample', //STRING, REQUIRED. THE CSS SELECTOR OF OUR TARGET
	xdata:'http://podaac-w10n.jpl.nasa.gov/w10n/allData/amsre/L3/sst_1deg_1mo/tos_AMSRE_L3_v7_200206-201012.nc/lon[]', //STRING, REQUIRED. A URL OF OUR X DATA
	ydata:'http://podaac-w10n.jpl.nasa.gov/w10n/allData/amsre/L3/sst_1deg_1mo/tos_AMSRE_L3_v7_200206-201012.nc/lat[]', //STRING, REQUIRED. A URL OF OUR Y DATA
	zdata:'http://podaac-w10n.jpl.nasa.gov/w10n/allData/amsre/L3/sst_1deg_1mo/tos_AMSRE_L3_v7_200206-201012.nc/tos[0:1]', //STRING, REQUIRED. A URL OF OUR Z DATA
	zmask:[100000002004087730000], //ARRAY, OPTIONAL. ANY VALUES YOU DON'T WANT PLOTTED GO HERE.
	zlabel:'zlabel', //STRING, OPTIONAL. A CUSTOM Z AXIS LABEL.
	ymask:[100000002004087730000], //ARRAY, OPTIONAL. ANY VALUES YOU DON'T WANT PLOTTED GO HERE.
	xlabel:'xlabel', //STRING, OPTIONAL. A CUSTOM X AXIS LABEL.
	ylabel:'ylabel', //STRING, OPTIONAL. A CUSTOM Y AXIS LABEL.
    xlimit:[225,280], //ARRAY, OPTIONAL. THE LIMITS (LOW,HIGH) OF THE X AXIS
    ylimit:[225,280], //ARRAY, OPTIONAL. THE LIMITS (LOW,HIGH) OF THE Y AXIS
	format:'spherical', //STRING, OPTIONAL. Possible values: sphericalterrain|spherical|flat|terrain (DEFAULTS TO FLAT).
	width:300, //INT OR STRING (I.E. '400px'), OPTIONAL. WILL DEFAULT TO PARENT CONTAINER SIZE IF NOT SPECIFIED
	height:200 //INT OR STRING (I.E. '400px'), OPTIONAL. WILL DEFAULT TO PARENT CONTAINER SIZE IF NOT SPECIFIED,
    textcolor:'#000', //hex color of our text, lines, etc. #000 (black) by default
    colorbar:['#fff','#e00','#ccc',...], //array of custom colorbins. defaults to rainbow
	rift:false, //BOOLEAN, OPTIONAL. USE THE OCULUS RIFT RENDERER, FALSE BY DEFAULT make sure to set your widget to be fullscreen in the CSS somewhere...
	rotate:false //BOOLEAN, OPTIONAL. ROTATE THE MESH IN THE SCENE, FALSE BY DEFAULT
}
